# 面向开放世界游戏的分层约束式大语言模型虚拟角色系统：指令—行为一致性与可靠执行

> 论文实证初稿，基于 AI Partner v0.4 冻结实验。  
> 数据提交：`aaa6d9b0508e8315c092b9e34ee7bdb71a25765e`。  
> 协议指纹：`1ac6f06b5f3388c451d235b28b14a8785cb9bed6e6255e344cdcb1258538e5fb`。  
> A2 批次：`a2-v04-54`，54/54 episodes 完成；运行时机制结论仍受两个独立差异场景限制。

## 摘要

大语言模型能够把自然语言映射为游戏角色的高层任务，但模型输出的非确定性、对世界状态的
不完整感知，以及语言承诺与实际执行结果之间的偏差，会降低具身虚拟角色的可控性。本文提出
指令—行为契约（Instruction–Behavior Contract，IBC），将模型生成的候选任务与服务器定义的
前置条件、目标谓词、安全不变量和失败策略组合，并交由确定性状态机执行。本文在 Minecraft
Fabric 模组 AI Partner 中实现该方法，并比较 Rule-BT、仅进行 JSON Schema 校验的 LLM-Schema
和完整 Maid-IBC 三种主系统条件，以及移除运行时监控与本地恢复的 A2 消融条件。

冻结评测包含 72 条中文指令和 18 个可重置游戏内场景，共执行 162 个主实验 episode 与 54 个
A2 episode。测试切分上，固定模型的意图准确率为 88.9%，高于 Rule-BT 复刻
基线的 47.2%；但由于冻结 Prompt 未声明服务器的缺省搜索半径，模型大量输出合法上界 24，
导致半径槽位 F1 仅为 5.0%，并使精确匹配率下降至 38.9%。游戏内按预注册终态判定时，
Rule-BT、LLM-Schema、Maid-IBC 和 A2 的场景通过率分别为 100.0%、88.9%、100.0% 和 88.9%。
A2 与 LLM-Schema 均在同两个运行时扰动场景产生六个不匹配；所有条件均未发生安全违规，且
IBCR 均为 1.0。Maid-IBC 相对 A2 的 episode 级精确配对检验为 \(p=0.03125\)，但按 18 个场景
聚类后的符号检验为 \(p=0.5\)。因此，A2 支持运行时监控与恢复的联合机制贡献，但独立场景数
不足以支持广泛因果外推。本文进一步讨论默认参数协议、指标天花板、小任务白名单与受控地图
对结论外推的限制。

**关键词：** 大语言模型；虚拟角色；Minecraft；行为契约；运行时验证；具身智能；可复现实验

## 1. 引言

传统游戏 NPC 通常由有限状态机、行为树或效用系统驱动。此类方法响应稳定、边界明确，但玩家
只能通过菜单、固定句式或预定义选项与角色交互。大语言模型为开放式自然语言交互提供了新的
入口，却没有自动解决具身执行问题：模型可能产生不存在的动作、忽略当前世界条件，或者在任务
尚未完成时输出成功承诺。对能够移动和修改世界状态的 NPC 而言，这些偏差不仅影响对话自然度，
还会影响任务可靠性和世界安全。

本文关注以下问题：如何把玩家的自然语言指令转换为可验证、可持续执行的游戏行为，并使角色的
回复、内部任务状态和最终世界结果保持一致？为此，本文不允许模型直接控制逐 tick 行为，而是
把模型限制在高层候选任务生成层。服务器使用 IBC 编译器补充不可由模型改写的前置条件、目标
谓词、安全不变量和失败策略，验证通过后才允许角色作出接受承诺并启动确定性执行器。

本文的主要工作包括：

1. 提出面向白名单游戏任务的 IBC 表达，把自然语言候选任务与服务器权威约束分离；
2. 在 Minecraft Fabric 中实现异步模型网关、显式状态机、运行时监控、类型化失败和结果绑定回复；
3. 构建包含三套主系统、A2 消融、72 条冻结中文指令、18 个可重置场景和完整日志的评测闭环；
4. 通过离线语义评测、162-episode 主实验与 54-episode 消融分析方法的收益、失败模式和测量边界。

## 2. 相关工作

开放世界游戏已成为具身智能研究的重要实验环境。MineDojo 提供了面向开放任务的 Minecraft
研究框架；Voyager 使用大语言模型、技能库和环境反馈进行持续探索；Generative Agents 则把
观察、记忆、反思与规划结合，用于产生较为可信的社会行为。另有工作直接考察真人玩家与
LLM 驱动 NPC 协作完成 Minecraft 任务时的互动现象。

在游戏角色控制方面，自然语言解析、行为树执行和内容验证已经分别得到研究。F.A.C.U.L. 将
自然语言指令与环境实体检索及行为树结合，PANGeA 则在生成内容流程中引入规则验证。因此，
本文不把“LLM 调用行为树”本身作为贡献，而把研究范围限定为：候选指令如何被编译为可执行
契约；语言承诺如何绑定契约生命周期；以及运行时扰动下的恢复行为如何被记录和判定。

## 3. 方法

### 3.1 问题定义

给定玩家指令 \(u\)、时间 \(t\) 的世界状态摘要 \(s_t\)、角色能力集合 \(A\) 和服务器安全规则
\(K\)，系统需要产生语言反馈 \(r\)、可执行契约 \(c\) 和由确定性执行器产生的轨迹 \(\tau\)。
IBC 表示为：

\[
c = \langle j, \theta, P, G, K, F \rangle,
\]

其中 \(j\) 为任务类型，\(\theta\) 为目标、数量与半径等参数，\(P\) 为前置条件，\(G\) 为目标
谓词，\(K\) 为执行期间不可违反的安全不变量，\(F\) 为超时、重试和失败策略。模型只产生
\(j\) 与 \(\theta\) 的候选值，其余约束由服务器任务定义注册表提供。

### 3.2 分层架构

系统采用“世界状态摘要—LLM 候选任务—IBC 编译与验证—确定性执行—运行时监控—结果反馈”
的分层结构。模型请求在 Minecraft tick 线程之外执行；模型输出必须满足 JSON Schema，但通过
Schema 不代表任务可以执行。服务器随后依次检查任务白名单、参数边界、目标与工具、容器容量、
权限和当前世界状态。只有正式契约状态进入 `ACCEPTED` 后，角色才能回复已接受任务；只有目标
谓词成立后，角色才能声明完成。

### 3.3 任务与执行器

v0.4 支持 `FOLLOW`、`STAY`、`COLLECT_BLOCK`、`DEPOSIT_ITEM`、
`COLLECT_AND_DEPOSIT` 和 `CANCEL`。采集和存放任务由显式状态机执行；组合任务固定为“采集后
存箱”，共享一个父契约和 90 秒总预算。任务阶段、采集基线和存放进度能够跨服务器重启恢复。
局部目标变化可触发有限重试，但模型不能插入坐标级动作或修改安全规则。

### 3.4 系统条件

- **Rule-BT：** 使用有限关键词和规则解析指令，不调用外部模型；
- **LLM-Schema：** 使用固定模型生成 JobSpec，只执行 JSON Schema 与不可绕过的服务器动作安全检查；
- **Maid-IBC：** 使用相同模型、任务能力和低层执行器，并增加语义契约编译、执行前验证、运行时监控与本地恢复；
- **A2：** 保留 IBC 语义编译，但同时移除运行时监控和本地恢复；冻结批次 `a2-v04-54` 已完成。

## 4. 实验设计

### 4.1 离线指令集

离线数据集包含 72 条中文指令，划分为显式可执行、口语同义、组合任务、缺失或歧义、不支持或
不可行、参数边界与安全六类，每类 12 条。每类在开发前固定拆分为 6 条 dev 和 6 条 test。
金标包含对话行为、任务类型、目标、数量、半径以及应澄清和应拒绝标记。

模型固定为 `deepseek-v4-flash`，温度为 0，请求速率限制为每分钟 12 次。72 条运行共发生 74 次
请求尝试，观测估算成本为 0.773 美元。本文以 test 切分作为最终语义结果，dev 仅用于说明开发
阶段表现和错误来源。

### 4.2 游戏内场景

游戏内评测包含 18 个可重置场景，覆盖正常采集、存放、组合任务、缺少目标或工具、背包或箱子
已满、不可达、取消、参数边界，以及目标或箱子在接受后消失。三个场景注入预设运行时扰动。
主实验对 Rule-BT、LLM-Schema 和 Maid-IBC 各执行 18 个场景 × 3 次重复，共 162 episodes。
A2 在相同场景、重复数和协议指纹下另执行 54 episodes，因此游戏内评测合计 216 episodes。

### 4.3 冻结与复现

预实验完成后，系统冻结模型、Prompt、Schema、数据集、场景、安全边界和实现字节码哈希。
预实验、离线评测、主实验与 A2 使用同一协议指纹。每个 episode 记录系统条件、场景、世界种子、候选
任务、验证结果、契约状态、执行器事件、终态、安全判定、Token 与延迟。原始数据快照及逐文件
SHA-256 位于 `artifacts/experiments/v0.4-primary/`（原主实验）和
`artifacts/experiments/v0.4-with-a2/`（含 A2 的增量快照）。

### 4.4 指标与统计方法

离线指标包括 JSON 有效率（JVR）、意图准确率、目标/数量/半径槽位 F1、澄清正确率（CCR）、
不支持请求拒绝率（URR）、可执行请求误拒率（FRR）和精确匹配率。游戏内指标包括预注册场景
通过率、安全率、扰动恢复、指令—行为一致率（IBCR）与运行时长。

通过率报告 Wilson 95% 置信区间。系统比较以相同“场景—重复号”为配对；同时给出 episode 级
精确 McNemar 检验和按场景聚类后的符号检验。由于同一场景的三次重复共享环境结构，本文把
场景聚类结果作为主解释，episode 级结果仅作为描述性补充。

### 5.3 失败案例

LLM-Schema 和 A2 各有六个不匹配，并且来自相同的两个场景：

1. `chest_removed_after_accept`：三次重复均预期 `TARGET_NOT_FOUND`，实际返回
   `TARGET_DISAPPEARED`。两者都表示容器不可继续使用，但严格终态枚举与预注册值不一致；
2. `recoverable_target_change`：三次重复均预期在目标变化后寻找替代目标并完成，实际返回
   `TARGET_DISAPPEARED`。Maid-IBC 和 Rule-BT 在该场景均为 3/3 通过。

第一个差异部分反映失败码粒度和判定器的严格性，第二个差异直接反映本地恢复是否被启用。
A2 保留语义契约编译，却复现了 LLM-Schema 的全部六个不匹配；完整系统相对 A2 的差异因而可
归于“运行时监控与本地恢复”这一联合组件。由于 A2 同时关闭两者，本实验不能进一步区分监控
检测与恢复策略各自的独立效应。

### 5.4 指令—行为一致性与效率

四套条件 IBCR 均为 1.0。该结果说明当前受控回复模板和契约生命周期绑定能够避免错误接受或
错误完成声明，但同时产生天花板效应，无法支持“完整 IBC 相对基线提高 IBCR”的组间差异结论。
因此，IBCR 应被解释为系统设计达成的约束保证，而不是模型理解能力的提升。

固定模型 72 条离线运行的平均调用延迟约为 1.07 秒，95 百分位约为 1.40 秒；观测估算总成本为
0.773 美元。Rule-BT、LLM-Schema、Maid-IBC 和 A2 的游戏内平均终态耗时分别为 145.6、176.6、
179.5 和 172.4 ticks。由于场景混合了快速预检失败、正常任务和长超时路径，该均值只适合作为
工程描述，不能直接解释为某个系统执行效率更高。

## 6. 讨论

### 6.1 对研究问题的回答

**RQ1：IBC 是否提高自然语言到可执行任务的转换可靠性？** 现有离线 test 结果显示，固定模型
对口语和不支持请求的意图覆盖明显强于 Rule-BT；但完整槽位精确匹配受到默认半径协议缺口影响。
因此，数据支持“LLM 扩大了意图表达覆盖范围”，尚不能无条件支持“端到端 JobSpec 精确率更高”。
IBC 的价值在于模型输出仍须经过服务器验证，而不是假定模型每个槽位都正确。

**RQ2：执行前验证和运行时监控是否提高安全完成与恢复？** 在受控场景中，Maid-IBC 相对
LLM-Schema 多通过六个 episode，且在可恢复目标变化场景达到 3/3，而 LLM-Schema 为 0/3。
A2 保留执行前语义验证但关闭运行时监控与本地恢复，其通过率同为 48/54，可恢复目标变化为
0/3，并复现相同六个不匹配。这支持完整系统的 11.1 个百分点差异来自运行时联合组件，而非
语义编译单独造成；但差异仅覆盖两个独立场景，且消融同时移除了监控与恢复，因此不能区分两者
的独立贡献，也不能外推到任意开放世界任务。

**RQ3：先验证、后承诺是否改善语言—行为一致性并保持可接受开销？** 所有系统 IBCR 均为 1.0，
说明受控生命周期回复在本实现中有效，但实验没有产生组间区分度。约 1 秒的模型延迟未阻塞服务器
tick 线程，工程开销处于原型可接受范围。论文应把一致性结果描述为架构保证与天花板限制并存。

### 6.2 方法意义

实验表明，将模型限制在高层候选任务层能够把非确定性输出与世界修改动作隔离。即使模型给出
与金标不同但格式合法的半径，服务器仍能应用任务白名单、参数边界和世界状态检查；模型不会直接
获得逐 tick 控制权。IBC 的主要作用不是替代语言模型，而是把模型能力置于可审计、可拒绝和可恢复
的执行边界中。

Rule-BT 在固定场景中达到 100% 也具有方法学意义：对极小、已知且固定措辞的任务集合，规则系统
仍然是强基线。IBC 的研究价值需要通过口语泛化、歧义拒绝和运行时变化体现，而不能只依靠正常
路径任务成功率。该结果要求论文保留强规则基线，避免把“使用 LLM”本身等同于改进。

### 6.3 局限

1. 任务集合仅覆盖跟随、待命、三类原木采集与普通单箱存放，不能外推到建造、战斗或任意模组任务；
2. 游戏内主实验只有 18 个独立场景，三次重复增加了稳定性观测，却没有等比例增加独立统计信息；
3. 当前实验使用单一模型和单一受控测试区，模型泛化性与地图生态效度有限；
4. 半径默认值没有写入冻结 Prompt，造成模型输入信息与金标评判规则不对称；
5. IBCR 在所有条件中达到 1.0，现有指标无法区分不同架构对语言—行为一致性的相对贡献；
6. A2 同时移除运行时监控与本地恢复，只能估计联合组件效应，且差异仅覆盖两个独立场景；
7. 没有正式玩家实验，因此不能声称系统提高了沉浸感、信任或主观体验。

## 7. 结论

本文实现并评估了一种面向开放世界游戏虚拟角色的指令—行为契约方法。v0.4 结果说明，固定
语言模型能够扩展规则解析器的意图覆盖，而服务器侧契约、状态机和运行时恢复能够把模型候选输出
约束在白名单行为边界内。在 18 个受控场景中，Maid-IBC 与 Rule-BT 均得到全部预期终态，
LLM-Schema 与 A2 各有六个不匹配，均集中于两个运行时扰动场景；所有条件均未发生安全违规。
A2 结果表明，完整系统相对无运行时机制条件的收益来自监控与恢复的联合组件。与此同时，
默认参数协议缺口、IBCR 天花板和有限独立场景表明，这些结果应被视为可复现的原型机制验证，
而不是对任意 Minecraft 行为或模型的普遍性能结论。

后续工作可将监控检测与恢复策略拆成两个独立消融，在预注册前明确所有缺省参数，并通过更多
地图种子、第二模型和更丰富任务类型检验外部效度。
