# AI Partner 0.12 架构

## 单一权威状态

女仆只有三个可持久化长期模式：

- `FOLLOW`：跟随主人，必要时使用原版安全传送；
- `STAY`：以当前位置建立两格待命锚点；
- `WORK`：按日程、地点和工作技能组合活动。

战斗、睡眠、回家、搜索、导航、执行和冷却都是瞬时阶段，不作为模式写入存档。旧存档的 `FOLLOWING`、`STAYING` 会迁移到对应模式，其他旧任务显示状态迁移为 `WORK`。

## Brain 仲裁

`MaidStateSensor` 把模式、日程、菜单暂停和工作导航接管状态投影到无 Codec 的瞬时记忆：

| 记忆 | 作用 |
| --- | --- |
| `FOLLOW_OWNER` | 启用跟随行为 |
| `STAY_IN_PLACE` | 清除移动意图并待命 |
| `PAUSED` | 菜单打开时暂停 |
| `WORK_CONTROLLED` | 工作状态机接管导航 |
| `SCHEDULE_WORK` / `SCHEDULE_REST` | 选择工作或休息 Activity |
| `AMBIENT_MOVEMENT` | WORK 模式的休闲移动 |
| `ACTIVITY_TARGET` | 回家、睡眠和日程地点 |

Activity 优先级为 `FIGHT → WORK → REST → IDLE`，`CORE` 始终运行。移动行为写入 `LOOK_TARGET` / `WALK_TARGET`，原版 Sink 和导航器负责实际路径。

## 狼式战斗与装备

`MaidThreatSensor` 不扫描无关生物，只读取：

1. 女仆自身最近攻击者；
2. 非 STAY 时主人的最近攻击者；
3. 非 STAY 时主人最近攻击的合法目标。

目标过滤复用原版狼的 Creeper、Ghast、已驯服动物、已驯服马与 PvP 排除规则，并额外遵守女仆活动边界。战斗控制器按距离选择弓或近战，装备控制器从主手与 35 格储物中选择合适武器。副手盾牌在近距离、可见威胁面前持续使用，攻击瞬间结束格挡后再出手。

装备控制器每秒比较一次四格护甲，使用原版 `Mob.canReplaceCurrentItem` 的护甲值、韧性、附魔与耐久规则；副手会自动装备具有 `blocks_attacks` 组件的盾牌。

## 基础技能

`MaidSkillType` 描述“会做什么”，不描述一次性目标。当前 29 项技能分为：

- 移动与拾取；
- 2×2 随身制作、3×3 工作台制作；
- 徒手、斧、镐、铲、锄、收获、种植、放置；
- 打开容器、取物、存物、记忆内容；
- 熔炉、高炉、烟熏炉；
- 剪毛、挤奶、喂养、照料、采蜜、钓鱼、灭火；
- 近战、远程战斗、盾牌格挡。

`MaidSkillSet` 聚合稳定的原子动作实现与最多 64 个容器的 LRU 观察记忆。`MaidWorkMode` 是技能组合与循环规则，例如：

- `LUMBERJACK`：导航 + 徒手破坏 + 斧头伐木；
- `MINER`：导航 + 镐子采矿；
- `SMELTER`：导航 + 熔炉 + 容器打开/取物/存物/记忆；
- `FISHER`：导航 + 钓鱼 + 拾取。

工作控制器使用 `搜索 → 导航 → 复验 → 动作 → 掉落回收 → 冷却` 的有界状态机。需要工具的规则通过装备租约把斧、镐、铲、剪刀、容器或鱼竿切到主手；退出动作时安全归还。所有世界修改遵守 `mobGriefing`。

## 物品栏和菜单

物品布局为：

- 原生主手 1 格；
- 储物 35 格；
- 原生护甲 4 格；
- 原生副手 1 格。

菜单只映射这 41 个槽与玩家 36 格背包，不再创建 2×2 输入/结果槽。自动制作仍由 `CraftItemAction` 的 `PERSONAL_2X2` 和 `WORKBENCH_3X3` 网格能力执行。

## 驯服、拴绳和前哨站

未归属女仆只接受 `ai-partner:maid_taming_foods` 标签中的安全食物，服务端消耗一份并按 1/3 概率驯服。其他普通交互落回原版 `TamableAnimal`，因此拴绳正常工作。

`ServerChunkEvents.CHUNK_GENERATE` 只在区块首次生成时识别有效的掠夺者前哨站结构起点，并且只登记结构起始区块。实际落地延迟到该世界的 tick 末尾：待结构周边区块全部可用后，在结构边缘搜索地势平整、25 个地板格都有自然支撑且不与任一原版结构部件相交的位置。崎岖地形会补齐地基，不再使用悬空回退。

笼子是新增的 5×5 前哨站风格建筑，不修改原版 Jigsaw 笼子池；其中只生成一只带持久标记、未绑定且处于 `STAY` 的女仆。由于入口是一次性的区块生成事件而不是生物刷新规则，区块卸载、重载和普通怪物刷新都不会增殖女仆。

## 持久化

实体存档写入模式、41 格物品布局、生活日程、工作配置、容器记忆、成长和皮肤哈希。Brain 瞬时记忆、当前路径、攻击目标、盾牌使用状态和工作扫描迭代器不写入存档，加载后由传感器与控制器重建。
