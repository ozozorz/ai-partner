# AI Partner v0.7 实机测试报告

> 测试日期：2026-07-22  
> 测试对象：AI Partner 0.7.0  
> 环境：Minecraft 26.1.2、Fabric Loader 0.19.3、Fabric API 0.155.2+26.1.2、Java 25

## 1. 自动化基线

- `gradlew test`：65 项单元测试全部通过；
- `gradlew build`：服务端、客户端、Mixin、资源处理与测试完整通过；
- 新增资源标识符命令回归测试，确认 `/maid collect minecraft:oak_log 1 8` 可以通过 Brigadier 解析。

## 2. 启动与基础控制

开发客户端成功加载 `ai-partner 0.7.0`。以下控制均在真实单人世界中执行并生效：

- `/maid spawn`；
- 全天工作日程、工作地点与工作半径设置；
- `/maid work <mode>` 与 `/maid combat <policy>`；
- `/maid status`、`/maid inventory`；
- `/maid collect minecraft:oak_log 1 8`。

## 3. 基础工作包验收

### 3.1 农业与工作边界

在工作区内放置成熟小麦并启用 `FARMER`：

- 女仆识别成熟作物、导航、收获并补种；
- 真实掉落物被跟踪并收入女仆背包；
- 背包最终包含小麦和小麦种子；
- 放在工作半径外的甘蔗保持不变，验证工作边界没有被越过。

### 3.2 有限采集与真实掉落

为女仆提供铁斧后执行正式采集命令：

```text
/maid collect minecraft:oak_log 1 8
```

女仆完成搜索、导航、破坏、掉落物跟踪和拾取，契约最终为 `COMPLETED`，背包包含 `minecraft:oak_log x1`。该过程没有直接伪造物品，也没有绕过原版掉落实体。

## 4. 防御中断与任务恢复

使用真实僵尸完成近战/弓箭和任务中断验收：

1. `DEFEND_OWNER` 能发现合法威胁并进入 `FIGHTING`；
2. 有弓和箭时会生成真实箭实体并消耗实际弹药，近距离或无远程条件时切换近战；
3. 战斗中启动有限采集任务，`/maid status` 同时显示 `Mode: FIGHTING`、`job: COLLECT_BLOCK`、`contract: RUNNING`；
4. 清除威胁后，女仆退出临时战斗中断并继续原任务；
5. 最终状态为 `Mode: WORKING`、`contract: COMPLETED`，背包包含采集到的原木。

这证明战斗层不会替换任务对象、契约或阶段，有限任务的超时也会在战斗期间暂停。

## 5. 测试边界与观察

- 无护甲、基础生命值的女仆可能在长时间面对高生命值或不可杀死目标时死亡；这符合当前“死亡即消失”和原版伤害边界，不属于无敌保护系统；
- 用于稳定观察中断状态的高抗性僵尸只是测试夹具，不是正常玩法假设；
- 离线开发环境产生了 Realms/身份验证网络警告，但未发现 AI Partner 引发的崩溃或异常；
- 本次为代表性真实世界验收，不等同于覆盖全部方块、实体和路径组合的自动 GameTest。

## 6. 结论

v0.7 的关键闭环已在真实游戏 tick 中成立：

```text
日程与工作地点约束
  → 有界搜索
  → 导航与动作前复验
  → 原版世界动作
  → 真实掉落物拾取
  → 战斗临时中断
  → 威胁解除后恢复原任务
```

因此 v0.7 可以作为后续 v0.8 整树砍伐、挖矿、熔炉与钓鱼工作包的稳定基础；尚未实现的能力不会在本版本中以占位或伪造结果冒充完成。
