# AI Partner 0.12 真实游戏测试报告

目标版本：Minecraft Java Edition 26.1.2、Fabric Loader 0.19.3、Fabric API 0.155.2

模组版本：AI Partner 0.12.0

测试世界：`AI Partner 0.12 Test`、`AI Partner Outpost Retest`

执行：

```powershell
.\gradlew.bat test compileClientJava --rerun-tasks --no-daemon
.\gradlew.bat build --no-daemon
```

结果：

- 主代码与客户端代码编译通过；
- 17 个测试套件、43 个 JUnit 测试全部通过；
- 0 失败、0 错误、0 跳过；
- 发布 JAR 可正常构建。

测试覆盖模式迁移、菜单动作与槽位布局、29 项技能定义、物品容量和持久化、制作规划、工作注册、熔炉批次守恒、自然树策略、钓鱼几何、成长与皮肤校验等纯逻辑。

## 2. 客户端启动、模式与菜单

通过 Fabric Loom `runClient` 启动真实 Minecraft 图形客户端并创建专用创造世界。

实测结果：

- 按 `R` 不再打开本地对话或任何模组界面；
- 连续执行两次 `/maid spawn` 均成功，聊天明确显示不设拥有数量上限；
- 女仆模型使用模组内置 `maid.png` 外观，不再回退到 Alex；
- 潜行右键打开菜单后，可见主手加 35 格储物、四格护甲和副手，共 41 个女仆槽位；
- 菜单只提供 Follow、Stay、Work 三种模式，三项均可即时切换；
- 菜单显示 29 项技能和容器记忆数量；
- 菜单中没有 2×2 制作格，也没有战斗策略选择控件。

## 3. 食物驯服与拴绳

生成一只无主女仆后进行隔离测试：

- 空手普通右键没有认主、模式查询或聊天输出；
- `/data get ... Owner` 返回找不到 Owner；
- 使用安全食物右键时，失败会显示烟雾粒子，成功会显示爱心并输出驯服成功消息；
- 成功前不会写入 Owner，成功后进入 Follow；
- 手持拴绳普通右键后，客户端可见玩家与女仆之间的真实拴绳线。

这验证了普通交互会在非驯服场景回落到原版可拴绳逻辑，而不是被模组右键菜单吞掉。

## 4. 装备选择、盾牌与狼式战斗

把钻石头盔、盾牌和钻石剑放入女仆储物格，约一秒后观察到：

- 钻石头盔自动移入原生头部护甲格并显示在模型上；
- 盾牌自动移入副手；
- 钻石剑自动移入主手；
- `/data get entity ... equipment` 同时返回 `diamond_helmet`、`shield` 和 `diamond_sword`。

随后生成固定敌对目标，由玩家先攻击目标触发“协助主人攻击”：

- 女仆立即接近目标；
- 接近过程中可见盾牌抬起；
- 进入攻击距离后切回钻石剑完成近战；
- 目标标签查询返回 `No entity was found`，确认真实伤害与击杀发生；
- 战斗不依赖任何可选战斗策略。

## 5. 生成蛋

执行 `/give @s ai-partner:ai_partner_spawn_egg` 后可获得带名称和图标的女仆生成蛋。对地面普通右键会生成一只新女仆实体，注册、资源模型和服务端实体创建均正常。

## 6. 掠夺者前哨站世界生成

真实运行在这一部分先后发现并修复了两个仅靠编译无法发现的问题：

1. 最初从 `CHUNK_GENERATE` 回调调用 `Level#setBlock`，会再次请求尚未完成的同一区块。Java 线程栈确认服务器线程停在 `ServerChunkCache#getChunk → OutpostMaidGeneration#buildCage`。最终改为只在区块生成回调登记前哨站，并在世界 tick 末尾、周边区块完成后落地笼子。
2. 第一版极端回退会把笼子放在结构最高点上方，实机全景显示为悬空。最终改为在原结构边缘搜索平整、25 个地板格均受自然地面支撑且不与任何原版结构部件相交的位置；崎岖地形会补齐地基。

最终在全新世界 `AI Partner Outpost Retest` 中定位前哨站 `[-224, ~, -416]`，观察到：

- 新笼子落在前哨站塔楼旁的地面上；
- 笼子与塔楼为两个独立结构，未覆盖塔楼或其结构部件；
- 笼中只有一只女仆，位置约为 `[-227.5, 64.0, -422.5]`；
- 女仆带 `ai_partner_outpost_maid` 持久标签；
- Owner 查询返回 `Found no elements matching Owner`，确认仍需玩家用食物驯服；
- 初次查询 UUID 为 `[I; 41730632, 271794933, -1647917915, -1973918939]`；
- 传送 1000 格使前哨站区块卸载，再返回后只输出同一个 UUID，未生成第二只女仆。

生成入口只监听结构起始区块的首次生成，不注册到生物刷新表；笼位还会逐一排除原版结构部件，因此不会替换悦灵或铁傀儡笼，也不会按怪物刷新规则增殖。
